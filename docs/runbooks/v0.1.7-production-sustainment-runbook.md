# Runbook: v0.1.7 Production Runtime Sustainment

This runbook provides operators with instructions to sustain, monitor, and troubleshoot the **v0.1.7** Swarm Relay & Computing production environment.

---

## 1. Monitoring the VPS Relay Worker (`HAS-WORKER-RELAY-001`)

The VPS relay status can be monitored either through local API proxy endpoints or directly on the VPS (`hoch-relay-001`).

### Checking via Local Proxy Endpoints
Retrieve the consolidated health, registry, and configuration status of the relay node:
```bash
curl -fsS http://localhost:8000/api/v1/relay/status | python3 -m json.tool
```
Expected output keys:
- `"worker_id"`: `"HAS-WORKER-RELAY-001"`
- `"worker_status"`: `"ONLINE"` (if Tailscale is connected) or `"UNKNOWN"` (if offline/unreachable)
- `"port_public_exposed"`: `false` (immutable security constraint)

---

## 2. Verifying Public Port Isolation

To guarantee that the VPS port `3012` is never exposed directly to the public internet, run the probe:
```bash
python3 -c "
import socket
s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
s.settimeout(3.0)
try:
    s.connect(('50.116.41.183', 3012))
    print('FAIL: PORT EXPOSED!')
    exit(1)
except Exception:
    print('PASS: PORT IS ISOLATED')
    exit(0)
"
```
Expected output: `PASS: PORT IS ISOLATED`

## 3. Database Maintenance (SQLite)

The local swarm ledger database `backend/swarm_ledger.db` contains tables for both accountability and doctrine storage.

### Verifying doctrine_rules Table
If the uvicorn logs report doctrine sync errors, verify the table exists and check rules loaded:
```bash
python3 scripts/verify_doctrine_db.py
```
To check entries directly:
```bash
sqlite3 backend/swarm_ledger.db "SELECT id, rule_text, source FROM doctrine_rules LIMIT 5;"
```

### Checking Agent Trust Scores
Verify the seeded trust scores for the relay worker node:
```bash
sqlite3 backend/swarm_ledger.db "SELECT * FROM agent_trust_scores WHERE agent_id = 'HAS-WORKER-RELAY-001';"
```

To execute a complete end-to-end check of both the local backend, doctrine storage, and VPS relay routing status, run:

All checks must output `[PASS]`.
