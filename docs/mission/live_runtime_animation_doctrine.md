# HOCH Agent Swarm Live Runtime Animation Doctrine

## Principle
Animations are not decorative. They are runtime telemetry.
Every animation must map to a real process, event, state transition, queue, model route, approval gate, or evidence artifact.

## Required Mapping
| Visual | Runtime Meaning |
|---|---|
| Koi swim path | Agent task moving through lifecycle |
| Koi color | Agent role or model provider |
| Koi speed | Queue pressure / execution latency |
| Ripple | Evidence ledger event committed |
| Glow pulse | Live health heartbeat |
| Orbit ring | Local model arbitration cycle |
| Amber pause | Approval required |
| Red stop | Blocked / failed / denied |
| Green stream | Local model success |
| Purple stream | Google/Gemini escalation review |
| Silver trail | Evidence archive / audit artifact |

## Forbidden
- Decorative-only process animations.
- Fake COMPLETE / GO states.
- Animation implying success before runtime evidence exists.
- Google/Gemini calls without escalation authorization.
- Human confirmation bypass for high-risk actions.
