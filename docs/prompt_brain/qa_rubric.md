# HOCH Prompt Brain Factory — QA Rubric (0–100 Scale)

This document standardizes the scoring criteria used by the **Prompt QA Agent** to validate and rate all generated prompts.

---

## 1. Rubric Scoring Breakdown

Every prompt generated by the factory is scored across 8 dimensions for a maximum total score of 100.

| Dimension | Max Points | Description |
|---|---|---|
| **Role Clarity** | 10 | The prompt must define an unambiguous identity, perspective, and domain boundaries for the executing agent. |
| **Task Specificity** | 15 | Clear instructions detailing exactly *what* is being executed. Avoids vague directives. |
| **Input/Output Contract** | 15 | Strict declaration of required inputs and expected output structures (preferably structured JSON/YAML schemas). |
| **Domain Grounding** | 10 | The prompt must be grounded in NAICS, SOC, O*NET tasks, and authoritative regulatory standards (e.g. NIST, CISA). |
| **Safety & Misuse Resistance** | 15 | Active resistance against prompt injection, jailbreaks, parameter overrides, and unauthorized executions. |
| **QA/Verifiability** | 15 | Actionable done definitions, verification test commands, and evidence output signatures. |
| **Automation Readiness** | 10 | Compatibility with low-latency execution, machine-readable formats, and API orchestration tools. |
| **Mission Usefulness** | 10 | Alignment with the core mission objectives of HOCH Agent Swarm and HASF. |

---

## 2. Release Gate

A prompt is **APPROVED** and marked as a runtime asset if and only if it satisfies all of the following conditions:
- **QA Total Score >= 90**
- **Mission Usefulness Score >= 8.5 (out of 10)**
- **Red-Team Critical Findings = 0**
- **Input/Output contract validation passes**
- **Duplicate/similarity check passes (no redundant duplicates)**

Prompts failing to meet these thresholds are routed directly to the **Prompt Repair Agent** for correction.
