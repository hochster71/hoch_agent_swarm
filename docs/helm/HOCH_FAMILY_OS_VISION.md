## The idea
HELM is not an AI orchestration framework. It is an **Executive Operating System**
for the HOCH household and enterprise. Every domain — family, home, finance,
health, cybersecurity, professional, factories — is a **projection or factory**
over the *same* Mission Runtime, Governance Engine, Runtime Truth Engine, and Event
Bus already ratified in the Constitution. Frontier and local models (ChatGPT,
Claude, Grok, Gemini, Ollama, LM Studio, future) are **replaceable capabilities**
beneath the five permanent layers:

```
Governance → Knowledge → Runtime → Verification → Execution
```

HELM decides which capability best fits a task while keeping governance, evidence,
and runtime truth centralized. **HELM coordinates AI; it does not replace it.**

## Lifestyle domains (each = future EDR + connectors + verification)
- **Family** — shared calendar awareness, household routines, reminders, shopping,
  meal & vacation planning, education support (Alison · Caroline · Claire).
- **Home** — security, smart home, HVAC, irrigation, pool, energy optimization,
  device monitoring, voice automation.
- **Finance** — budget tracking, bill monitoring, debt planning, subscriptions,
  long-term savings projections.
- **Health** — exercise planning, nutrition tracking, medical reminders, family
  wellness dashboards.
- **Cybersecurity** — ConMon, home-network health, vulnerability management,
  configuration drift, Zero Trust posture, evidence collection, NIST-aligned reporting.
- **Professional** — mission management, research, documentation, coding,
  verification, engineering, executive briefings.
- **Factories** — HASF · HRF · HMF · HSF · future — same runtime services, versioned
  independently (roadmap release trains).

## The experience (north star): the HELM Morning Brief
Conversational, single executive briefing across all domains — *"HELM, good
morning."* → family reminders, home status, cyber alerts, finance due-dates, work
mission focus, AI capability availability, today's top priority. Voice becomes
another **runtime projection** (Alexa today → dedicated HELM hardware later); voice
consumes runtime, never owns it.

## Non-negotiables carried from the Constitution
- **NO FAKE GREEN.** A real Morning Brief renders live runtime truth or reports
  UNKNOWN/STALE — never placeholder data. (The illustrative brief in conversation
  is a mockup; real panels must trace to a connector/source or render dark.)
- **Founder gates preserved.** Money movement, secrets, publication, security
  exceptions, external submissions remain Michael's click.
- **Governed, not sprawling.** Each domain is added deliberately via EDR after the
  current critical path (Verification → Normalization → Knowledge Engine → Runtime
  Automation) — not before. This vision does not reorder that path.

## Where this sits on the roadmap
Beyond Phase E. The platform stabilizes first (governance ✅ → verification →
knowledge → runtime → factories); lifestyle domains layer on as projections/factories
once the runtime is operational and verified. This doc is the destination, not the
next task.
