# Michael Prompt Quality Rubric

Every command and mission prompt entered into the system is graded based on the following dimensions:

1. **Scope clarity**: Specific paths, files, and classes to edit.
2. **Non-goals defined**: Explicit boundaries on what NOT to change.
3. **Runtime truth requirements**:live validation needs.
4. **Gate requirements defined**: Check scripts explicitly listed.
5. **Evidence required**: Reference evidence log requirements.
6. **Final report schema defined**: Output fields constraint.
7. **Anti-fake claim controls**: Prevention of fake PASS statuses.
8. **Human loop reduction**: Specific fallback logic if blocked.
9. **Rollback stop conditions**: Conditions to abort.
10. **Integration safety**: pytest and playwright coverage.
