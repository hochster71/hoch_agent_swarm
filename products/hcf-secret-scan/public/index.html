Secret & Key Exposure Scan

Paste a config, .env, log, or code snippet. An offline heuristic scan flags credential-shaped patterns — vendor key formats, decoded JWT structures, private-key blocks, connection-string passwords, high-entropy secret-named values. Matched values are always masked. Nothing you paste is stored.

HEURISTIC PATTERN SCAN — NOT A GUARANTEE. This is string-pattern matching over the exact text you paste, done server-side and discarded. It cannot tell whether a match is real, active, or sensitive, and a clean result does not mean the text contains no secrets. Nothing here is a security decision or advice.

What the paid report adds
  • Every matched pattern with its line number and masked value (originals are never echoed back).
  • What the pattern's documented format is — and the ordinary, benign explanation for each match.
  • Placeholder/low-entropy matches that were set aside, and why.
What it deliberately is not
  • No network calls, no vendor validation, no storage of your text.
  • No certainty claims, no "you are safe / you are breached", no advice.
