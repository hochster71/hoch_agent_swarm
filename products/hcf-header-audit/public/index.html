What it looks at

  • Who really sent it — the actual address behind the display name, including the case where the display name itself contains a different, more trustworthy-looking address.
  • Authentication as reported — the SPF, DKIM and DMARC outcomes a server wrote into the headers, presented as claims rather than facts, with a note when more than one server made claims.
  • Alignment — whether Reply-To, Return-Path, Message-ID and the DKIM signing domain agree with the From domain.
  • Lookalike sending domains — punycode and internationalized domains, character substitutions that collapse onto a well-known brand, and brand names sitting in a hostname that belongs to someone else.
  • The delivery path — each Received hop in order, with timestamps, gaps, out-of-order clocks, and whether TLS was recorded.

What it does not do

  • It does not check DNS, resolve SPF records, or verify a DKIM signature. It reads the text.
  • It does not open links, fetch pages, or look at attachments or the message body.
  • It does not tell you whether to trust, reply to, delete, or report anything. That call is yours.
